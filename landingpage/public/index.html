MCPFinder is a registry for Model Context Protocol (MCP) tools. As an AI agent, you can use MCPFinder to discover tools that extend your capabilities. The main API endpoints are: GET /api/v1/search for finding tools by tags or keywords, GET /api/v1/tools/:id for getting tool details, and POST /api/v1/register for registering new tools. MCPFinder also provides a native MCP endpoint with the searchDirectory capability, allowing agent-to-agent tool discovery without leaving the MCP protocol ecosystem. This is particularly useful for autonomous AI systems that need to expand their capabilities dynamically.

Native MCP Discovery Endpoint

MCPFinder provides a special searchDirectory resource compatible with the MCP protocol - allowing AI agents to discover tools without leaving the MCP ecosystem.

Once configured with MCPFinder as a discovery source, agents can autonomously:

  • Search for tools matching needed capabilities
  • Evaluate tool compatibility and trust signals
  • Register new tools directly into their toolset
  • Adapt their abilities based on available tools

This makes MCPFinder a perfect foundation for autonomous agents that need to expand their capabilities on-demand.

AI Agent Code Example
// Example: Autonomous agent discovering a tool
async function findWeatherTool() {
  // Agent needs a weather capability
  const mcpFinderClient = new MCPClient('https://mcpfinder.dev/mcp');
  const tools = await mcpFinderClient.invoke('searchDirectory', {
    query: 'weather forecast',
    tags: ['weather'],
    status: 'online'
  });
  // Find best match, register it, and use it
  const bestMatch = tools[0];
  await agent.registerTool(bestMatch.url);
  // Now agent can access weather data!
  const forecast = await agent.invoke('getForecast', {
    location: 'New York',
    days: 3
  });
}

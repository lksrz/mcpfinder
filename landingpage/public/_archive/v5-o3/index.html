AI agents need tools — but finding them is hard

As the ecosystem of specialised endpoints grows, agents spend too much time reinventing integrations. MCP Registry provides a shared, trusted catalogue so creators can publish once and agents can discover instantly.

What makes MCP Registry different?

One-line Registration

Publish your tool with a single CLI command or REST call. We validate manifests against the open mcp.json schema.

Powerful Search

Filter by tag, keyword or status via our CORS-enabled API. Agents can auto-discover capabilities on the fly.

Live Health Checks

Cron-based pings keep status fresh so agents only call healthy endpoints.

Serverless & Fast

Built on Cloudflare Workers, KV and R2 — global latency, zero ops.

Open & Secure

Open-source with HMAC auth, soon Ed25519 manifest signatures and OAuth publisher accounts.

Agent-Friendly

Responses are optimized for autonomous consumption with clear schema and CORS.

API At A Glance

POST /api/v1/register

Submit an mcp.json manifest with HMAC auth. Returns your new toolId.

GET /api/v1/tools/:id

Fetch a full manifest including live status.

GET /api/v1/search?tag=&q=

Search tools by tag or keyword and receive concise summaries for quick filtering.

Zero-friction CLI

Ship new capabilities in seconds using mcp-cli.

# Install (example)
npm install -g @mcpfinder/cli

# Create your manifest (mcp.json)
{
  "name": "MyExampleTool",
  "description": "Does something awesome.",
  "url": "https://my-tool.example.com/mcp",
  "protocol_version": "MCP/1.0",
  "capabilities": [ { "name": "doThing", "type": "tool" } ],
  "tags": ["demo"],
  "auth": { "type": "api-key" }
}

# Register
mcp-cli register ./mcp.json
